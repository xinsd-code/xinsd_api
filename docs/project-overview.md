# API Forge — 项目介绍文档

> 面向研发与数据团队的全栈 API 研发辅助平台  
> 集 Mock 管理、接口转发、数据编排、自然语言取数与多智能体 DB 问答于一体

---

## 一、项目定位与价值

API Forge 是一个面向研发和数据团队的一站式 API 研发辅助平台。它将日常开发中最常见的接口管理、数据联调、智能取数等场景整合为统一的工作台，让团队在一个平台内完成从接口 Mock → 接口联调 → 数据编排 → 自然语言取数的完整工作流。

**核心价值主张**

| 场景 | 传统方式 | API Forge |
|------|---------|-----------|
| 接口联调 | 依赖后端 Ready，等待部署 | Mock 工作台即开即用，前后端并行 |
| 数据格式适配 | 手写转换层代码 | 可视化编排 + AI Chat 驱动，零代码完成 |
| 临时取数 | 写 SQL → 找 DBA 审批 → 拿到密码 → 连数据库 | 用自然语言在工作台直接提问 |
| 数据分析 | 写 SQL → 手动整理结果 | AI 自动生成 SQL、执行、解读、追问 |

## 三、核心能力详解

### 1. 🔀 API 高级编排（Advanced Orchestration）

#### 概述

高级编排是 API Forge 的核心数据处理能力。它在 **API 转发** 的基础上，提供了一套可视化的数据处理流水线。开发者可以通过拖拽、配置节点的方式，将接口响应数据进行实时加工后再返回给调用方——无需编写任何代码。

#### 工作原理

```
客户端请求 /forward/<path>
        │
        ▼
  路由匹配（path-to-regexp）
        │
        ▼
  参数绑定 & 变量替换
        │
        ▼
  HTTP 转发至目标接口
        │
        ▼
  获取响应数据 ──────────────────────────┐
        │                              │
        ▼                              │
  检测 orchestration 配置               │
   ├─ 没有编排配置 → 直接返回原始数据     │
   └─ 存在编排配置 → 进入编排管道 ←──────┘
        │
        ▼
  [Orchestration Engine]
   filter → map → compute → sort
   （按 order 字段排序依次执行）
        │
        ▼
  返回处理后的数据
```

#### 四类处理节点

| 节点类型 | 功能说明 | 典型场景 |
|---------|---------|---------|
| **filter（筛选）** | 保留（include）或排除（exclude）指定字段 | 裁剪不必要字段，精简响应体 |
| **map（映射）** | 字段重命名、路径映射 | 统一前后端命名规范 |
| **compute（计算）** | 模板表达式计算，新增派生字段 | 生成折扣价、汇总指标 |
| **sort（排序）** | 数组排序 + 结果数量限制 | 按指标取 TopN |

#### 核心特性

- **嵌套路径支持**：使用 `data[].age` 语法直接操作对象数组内的属性
- **表达式引擎**：`{{field}}` 引用数据字段，`{{$param.key}}` 引用接口入参，支持四则运算
- **父子联动**：`map` 节点重命名父路径后，后续节点自动跟随新命名空间
- **AI Chat 驱动**：通过自然语言直接生成或修改整条编排链路；支持"修复体检报错"模式
- **单节点调试**：可对任意节点单独调试，实时查看中间输出状态
- **配置持久化**：保存后自动写入 `orchestration` 字段，运行时直接通过 `/forward/...` 访问

#### 编排节点配置示例

```json
{
  "nodes": [
    {
      "id": "filter_1", "type": "filter", "order": 0,
      "config": {
        "mode": "include",
        "fields": ["code", "data[].id", "data[].name", "data[].price"]
      }
    },
    {
      "id": "map_1", "type": "map", "order": 1,
      "config": {
        "mappings": [{ "from": "data[].price", "to": "amount" }]
      }
    },
    {
      "id": "compute_1", "type": "compute", "order": 2,
      "config": {
        "computations": [{
          "field": "data[].discountPrice",
          "expression": "{{amount}} * 0.9"
        }]
      }
    },
    {
      "id": "sort_1", "type": "sort", "order": 3,
      "config": {
        "arrayPath": "data",
        "sortField": "discountPrice",
        "order": "desc",
        "limit": 10
      }
    }
  ]
}
```

### 2. 🗣 NL2DATA — 自然语言取数工作台

#### 概述

NL2DATA 让非技术人员也能通过自然语言提问，直接从数据库获取所需数据。它采用两阶段 AI 链路设计（NER 识别 + SQL 生成），将用户的自然语言问句转化为精确的数据库查询语句，并在同一页面完成执行、查看与历史管理。

#### 两阶段链路

```
用户提问："查看近7天各渠道的订单金额"
  │
  ▼
┌──────────────────────────────────────┐
│  第一阶段：NER 名词识别               │
│                                      │
│  • 从语义配置中召回 NER 候选语料      │
│    （按关键词评分 → 去重 → TopN 截断）│
│  • AI 识别业务名词并映射到 table.column│
│  • 输出：matchedMetrics（渠道→channel, │
│    订单金额→order_amount）            │
│    normalizedTerms / timeHints（近7天）│
└─────────────┬────────────────────────┘
              │
              ▼
┌──────────────────────────────────────┐
│  第二阶段：SQL 生成                   │
│                                      │
│  • 将 NER 结果 + schema 摘要注入 Prompt│
│  • AI 生成参数化查询 SQL              │
│  • 通过只读校验 + 敏感字段阻断后执行  │
└─────────────┬────────────────────────┘
              │
              ▼
        结果表格展示
        + 会话历史持久化（SQLite）
```

#### 工作台功能

- **并列布局**：SQL 编辑区 / 会话历史面板 / 数据结果表格
- **会话持久化**：每轮对话记录用户问句、生成 SQL、完整 Prompt 到 SQLite
- **历史回溯**：点击历史条目查看完整上下文，一键"同步 SQL"回填到编辑区
- **去重执行**：重新执行已有 SQL 不会产生重复历史条目
- **安全校验**：只读关键字检查 + 危险关键字阻断 + 注释注入防护
- **多数据源支持**：支持 MySQL、PostgreSQL、MongoDB

### 3. 🤖 DB Harness — 多智能体数据问答

#### 概述

DB Harness 是 API Forge 中最复杂、最强大的子系统。它面向数据分析场景，构建了一套五阶段多智能体（Multi-Agent）对话链路，自动完成从自然语言理解到 SQL 生成、安全执行、结果解读的完整闭环。

与 NL2DATA 相比，DB Harness 提供了更完整的工作空间管理、知识记忆、智能追问和结果解读能力。

#### 五阶段 Agent 链路

```
用户提问
  │
  ▼
┌────────────────────────────────────────────────────────┐
│  Intent Agent — 意图理解                                │
│  理解用户意图（查询/对比/分析/诊断/可视化）              │
│  提取规划提示：维度、指标、筛选条件、时间范围             │
│  Fallback：模型不可用时，通过关键词匹配生成基础规划       │
└────────────────┬───────────────────────────────────────┘
                 │ PlanningHints
                 ▼
┌────────────────────────────────────────────────────────┐
│  Schema Agent — NER 字段映射                            │
│  将业务术语（"订单金额"）映射到数据字段（order_amount）   │
│  输出 matchedMetrics + normalizedTerms + timeHints     │
│  支持独立 NER 模型配置                                  │
│  Fallback：基于候选集评分 + 关键词匹配生成映射           │
└────────────────┬───────────────────────────────────────┘
                 │ NerPayload
                 ▼
┌────────────────────────────────────────────────────────┐
│  Query Agent — 查询计划生成                             │
│  生成结构化 QueryPlan（维度/指标/筛选/排序/LIMIT）       │
│  编译为参数化 SQL（防注入）                              │
│  支持 Prompt 自适应压缩（standard→compact→minimal）     │
│  Fallback：规则引擎生成基础 SQL                         │
└────────────────┬───────────────────────────────────────┘
                 │ CompiledSQL
                 ▼
┌────────────────────────────────────────────────────────┐
│  Guardrail Agent — 只读执行网关（安全核心）              │
│  三重安全校验：                                         │
│    ① 只读关键字校验（SELECT/WITH/SHOW/DESC/EXPLAIN）    │
│    ② 危险关键字阻断（INSERT/DROP/ALTER...）             │
│    ③ 敏感字段检测（password/token/email/phone/身份证）  │
│  Schema 合法性校验：表名/字段名存在性验证               │
│  参数化执行 + 行数限制包裹（防全表扫描）                 │
└────────────────┬───────────────────────────────────────┘
                 │ ExecutionResult
                 ▼
┌────────────────────────────────────────────────────────┐
│  Analysis Agent — 结果解读                              │
│  对查询结果进行自然语言解读                              │
│  生成结果摘要 + 3 条追问建议                             │
│  空结果智能提示（放宽时间 / 减少条件 / 换维度）          │
└────────────────────────────────────────────────────────┘
```

#### 工作空间与会话管理

- **多 Workspace**：每个 Workspace 绑定独立的数据源 + 工作规则（自定义 Prompt 约束）
- **多 Session**：每个 Workspace 下可维护多条独立对话会话
- **完整 Trace**：每轮对话记录完整的五阶段链路中间产物，前端可展开查看

#### 知识记忆系统

- 用户对回答点赞（positive）或纠错（corrective）后，记忆写入 SQLite
- 后续相关问题优先召回历史知识，辅助 Schema / Query Agent 提升准确率
- 知识条目按关键词评分排序，每次最多召回 24 条

#### 空结果自动重试

- 当查询返回 0 行且时间范围 ≤ 30 天时，自动放宽时间范围（×3，上限 3650 天）
- 重新执行 Query → Guardrail 链路，有数据则替换原结果
- 功能可通过环境变量 `DB_HARNESS_EMPTY_RESULT_RETRY` 开关

#### GEPA 离线评估系统

DB Harness 还内置了 **GEPA（Generative Evaluation and Policy Adaptation）** 离线评估子系统：

- 基于历史会话或合成样本，自动对比不同 Prompt 策略与 NER 参数的效果
- 生成候选排序报告与评分卡（成功率、空结果率、延迟、Token 开销）
- 审核通过后的最优候选配置可一键应用到 Workspace 运行时参数

## 四、NL2DATA 与 DB Harness 对比

| 维度 | NL2DATA | DB Harness |
|------|---------|------------|
| 目标用户 | 快速取数、轻量使用 | 深度数据分析 |
| Agent 数量 | 2（NER + SQL） | 5（Intent / Schema / Query / Guardrail / Analysis） |
| Workspace 管理 | 无 | 有（多 Workspace + 多 Session） |
| 知识记忆 | 无 | 有（Feedback → Knowledge Memory） |
| Catalog 快照 | 无 | 有（实体关系 + 语义分类） |
| 结果解读 | 无（仅返回 SQL + 数据） | 有（自然语言回复 + 追问建议） |
| 空结果处理 | 基础提示 | 自动重试 + 智能提示 |
| 安全校验 | 基础只读检查 | 三重校验 + Schema 合法性 + 敏感字段检测 |
| 数据源支持 | MySQL / PostgreSQL / MongoDB | MySQL / PostgreSQL |
| Fallback 机制 | 有（规则引擎降级） | 有（三级 Fallback：Intent / NER / SQL） |
| 离线评估 | 无 | 有（GEPA 系统） |

## 五、其他功能模块

### Mock 接口工作台

快速创建本地联调接口，支持 RESTful 动态路径、流式响应、JSON Body 匹配、分组变量管理。

### API 接入 (Client)

类 Postman 的接口调试工具，内置代理解决跨域问题，支持 JSON Body 表单化编辑。

### DB API（SQL 接口化）

将常用只读 SQL 封装为可复用的标准接口（`/query/...`），支持 `{{variable}}` 占位符自动识别与入参绑定，支持 Redis 结果缓存。

### 数据库实例管理

统一管理 MySQL / PostgreSQL / Redis / MongoDB 实例的连接信息，包含结构浏览与语义配置功能。

### 模型管理

管理 OpenAI Compatible 的 AI 模型来源，支持 Chat / Embedding 两类，支持实时校验 Model ID 可用性。

### 语义配置

连接数据库 schema 与 AI 理解能力的核心桥梁。支持图形化逐字段编辑语义名称、描述、别名、语义角色、计算口径、NER 开关，保存后自动派生所有消费链路所需的字段映射。
