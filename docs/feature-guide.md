## 1. Mock 接口工作台

**功能定位**：前后端并行开发的接口模拟工具

### 功能清单

| 功能 | 说明 |
|------|------|
| 创建 Mock 接口 | 定义 HTTP 方法、路径、响应状态码、响应体 |
| RESTful 动态路径 | 支持 `/users/:id` 等参数化路径 |
| 流式响应 | 支持 SSE/Streaming 类型的 Mock 响应 |
| JSON Body 匹配 | 根据请求体内容返回不同 Mock 数据 |
| 分组变量管理 | 接口按业务线分组，组内共享变量（如 baseUrl、Token） |
| 请求头 / 参数配置 | 可预设请求头、查询参数、请求体 |

### 使用场景

- 前端开发时后端接口尚未 Ready，先用 Mock 联调
- 需要模拟不同响应场景（成功、失败、超时等）
- 团队共享统一的 Mock 接口规范

### 访问路径

- 管理页面：`/`（首页）
- 运行时调用：`http://localhost:3000/mock/<path>`

## 3. API 转发 & 高级编排

**功能定位**：虚拟接口层 + 可视化数据处理流水线

### 3.1 API 转发

#### 功能清单

| 功能 | 说明 |
|------|------|
| 虚拟路径映射 | 建立 `/forward/<path>` 路径，映射到 Mock / API Client / DB API |
| 参数绑定 | 支持"接口入参绑定"与"固定静态值"两种模式 |
| Body 参数注入 | 目标为 POST application/json 时，可将入参写入请求体字段 |
| Redis 缓存 | 支持结果缓存，Key 规则支持 `{{param}}` / JSONPath 动态拼接 |
| 多目标类型 | 目标可选 Mock 接口、API Client 接口、DB API 接口 |

### 3.2 高级编排

#### 功能清单

| 功能 | 说明 |
|------|------|
| 可视化工作流 | 拖拽式节点编排画布 |
| filter 节点 | 字段筛选（保留 / 排除模式） |
| map 节点 | 字段重命名 / 路径映射 |
| compute 节点 | 模板表达式计算新增字段 |
| sort 节点 | 数组排序 + 结果数量限制 |
| 嵌套路径 | 支持 `data[].field` 操作数组内元素 |
| 表达式引擎 | `{{field}}` 引用数据字段，`{{$param.key}}` 引用入参 |
| AI Chat 编排 | 自然语言描述需求，AI 自动生成编排节点 JSON |
| 体检修复模式 | AI 自动修复当前编排配置中的错误 |
| 单节点调试 | 断点式执行，查看任意节点的中间输出 |
| 编排状态标记 | 显示"已编排" / "未编排"状态 |

#### 表达式语法

| 语法 | 示例 | 说明 |
|------|------|------|
| `{{field}}` | `{{price}} * 0.9` | 引用当前对象的字段值 |
| `{{$param.key}}` | `{{$param.taxRate}} * {{total}}` | 引用接口调用时的入参 |
| 四则运算 | `{{a}} + {{b}} * 2` | 支持 `+` `-` `*` `/` `()` |

### 访问路径

- 管理页面：`/api-forward`
- 运行时调用：`http://localhost:3000/forward/<path>`

## 5. NL2DATA 自然语言取数工作台

**功能定位**：自然语言驱动的快速取数工具

### 功能清单

| 功能 | 说明 |
|------|------|
| 自然语言提问 | 输入业务问题，AI 自动生成 SQL |
| 两阶段链路 | NER 名词识别 → SQL 生成，分步推理 |
| 数据源选择 | 从已配置的数据库实例中选择目标数据源 |
| 模型选择 | 选择用于推理的 AI 模型 |
| SQL 编辑区 | 可手动调整 AI 生成的 SQL |
| 数据结果表格 | 表格化展示查询结果 |
| 会话历史 | 持久化到 SQLite，包含问句 / SQL / Prompt |
| 历史回溯 | 点击历史记录查看完整上下文 |
| SQL 回填 | 一键将历史 SQL 同步到编辑区 |
| 去重检测 | 重复执行已有 SQL 不创建新历史条目 |
| 安全校验 | 只读校验 + 危险关键字阻断 + 注释注入防护 |
| 多引擎支持 | MySQL / PostgreSQL / MongoDB |
| Fallback 降级 | 模型不可用时降级为规则引擎生成 SQL |
| Trace 追踪 | 展示五阶段执行链路及状态 |

### NER 候选集构建规则

1. 从语义配置中提取所有启用 `enableForNer` 的字段
2. 按关键词相关度评分（字段名 ×2 + 指标名 ×2 + 别名 ×2 + 描述 ×1）
3. 按 Hard Limit（16 条）截断，避免 Prompt 过长

### 访问路径

- 工作台页面：`/nl2data`

## 6. DB Harness 多智能体数据问答

**功能定位**：面向数据分析场景的完整多智能体问答系统

### 功能清单

#### 6.1 对话核心

| 功能 | 说明 |
|------|------|
| 五阶段 Agent | Intent → Schema → Query → Guardrail → Analysis |
| 意图识别 | 自动识别查询 / 对比 / 分析 / 诊断 / 可视化意图 |
| NER 字段映射 | 业务术语精确映射到数据库表.字段 |
| 结构化查询计划 | 维度 / 指标 / 筛选 / 排序 / LIMIT 结构化输出 |
| 参数化 SQL | 编译为防注入的参数化查询 |
| 结果解读 | AI 生成自然语言回复，包含数据洞察 |
| 追问建议 | 每轮对话生成 3 条智能追问方向 |
| 空结果自动重试 | 自动放宽时间范围重试（可配置开关） |

#### 6.2 工作空间管理

| 功能 | 说明 |
|------|------|
| 多 Workspace | 每个 Workspace 绑定独立数据源 + 工作规则 |
| 工作规则 | 自定义 Prompt 约束，控制 AI 行为 |
| 多 Session | 每个 Workspace 下多条独立会话 |
| 运行时配置 | 可调节压缩级别、NER 候选上限、Schema 摘要表数 |

#### 6.3 知识记忆

| 功能 | 说明 |
|------|------|
| 正反馈标记 | 对好的结果点赞，记录"可复用"知识 |
| 纠错反馈 | 对错误结果提交纠错，记录"需优先纠偏"知识 |
| 知识召回 | 后续相关问题优先召回历史知识辅助推理 |
| 知识存储 | 持久化到 SQLite，按 databaseInstanceId 分组 |

#### 6.4 Trace 链路追踪

| 功能 | 说明 |
|------|------|
| 五阶段状态 | 每个 Agent 显示 pending / running / completed / failed |
| 中间产物 | 可展开查看各 Agent 的输入输出（JSON 格式） |
| Handoff 传递 | 显示每个 Agent 传给下一阶段的上下文内容 |

#### 6.5 安全保障

| 校验层 | 规则 |
|--------|------|
| 只读关键字 | 仅允许 `SELECT / WITH / SHOW / DESC / EXPLAIN` |
| 危险关键字 | 阻断 `INSERT / UPDATE / DELETE / DROP / ALTER / TRUNCATE / CREATE / GRANT / REVOKE` |
| 注释注入 | 阻断 `--` 和 `/* */` 注释 |
| 敏感字段 | 检测 password / token / email / phone / 身份证 等字段 |
| Schema 合法性 | 校验表名、字段名是否在实际 schema 中存在 |
| 行数限制 | 自动包裹子查询限制返回行数（最大 200 行） |
| 参数化执行 | 绑定参数防止 SQL 注入 |

#### 6.6 GEPA 离线评估

| 功能 | 说明 |
|------|------|
| 创建评估 Run | 基于 Workspace 历史会话或合成样本 |
| 候选策略 | 自动生成 Prompt 策略候选 + NER 参数候选 |
| 离线回放 | Heuristic 模式评估各候选的规划成功率与 Token 开销 |
| 评分卡 | 输出成功率、空结果率、平均/P95 延迟、Token 成本 |
| 候选排名 | 按综合分排序，推荐最优候选 |
| 一键应用 | 审核通过后将最优配置写入 Workspace 运行时参数 |

#### 6.7 Fallback 机制

当 AI 模型不可用时，系统降级到规则引擎：

| Agent | Fallback 策略 |
|-------|--------------|
| Intent Agent | 正则匹配意图（对比/趋势/诊断/查询），提取时间范围 |
| Schema Agent | 基于关键词评分在 schema + 语义配置中匹配候选指标 |
| Query Agent | 评分选表 → 识别聚合方式 → 自动选维度/指标/时间列 → 生成基础 SQL |

### 访问路径

- 问答工作台：`/db-harness`

## 9. 语义配置

**功能定位**：连接数据库 Schema 与 AI 理解能力的核心桥梁

### 功能清单

| 功能 | 说明 |
|------|------|
| 语义名称 | 为字段设置业务含义名称（如 `order_amount` → "订单金额"） |
| 字段描述 | 添加字段的业务解释 |
| 别名列表 | 设置多个别名用于 NER 匹配（"订单额"、"交易金额"等） |
| 语义角色 | 标注为 metric / dimension / time / identifier / attribute |
| 计算口径 | 说明字段的计算方式与统计规则 |
| NER 开关 | 控制是否纳入自然语言识别的候选集 |
| 自动派生 | 保存后自动生成所有消费链路所需的规范化映射 |

### 消费方

| 消费模块 | 使用的语义信息 |
|----------|--------------|
| DB Harness / Intent Agent | Schema 摘要、SemanticSnapshot |
| DB Harness / Schema Agent | NER 候选集（enableForNer 字段） |
| DB Harness / Query Agent | 指标-维度分类、外键关系 |
| NL2DATA | NER 候选集、Schema 摘要 |
| DB API SQL 编辑器 | 字段 Hover 提示 |
| 数据库实例详情 | 字段语义显示 |

### 配置层次

```
数据库实例
  └── 表（table）
        └── 字段（column）
              ├── metricName     → 业务指标名称
              ├── description    → 字段语义描述
              ├── aliases        → 别名列表
              ├── semanticRole   → 语义角色
              ├── calcMode       → 计算口径说明
              └── enableForNer   → 是否纳入 NER 候选集
```
