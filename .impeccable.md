## Design Context

### Users
这个产品主要面向后端开发、数据分析和平台研发同学，使用场景集中在本地联调、数据探索、接口封装前的验证，以及把数据库能力快速包装成可复用的应用层能力。

当前新增的 `NL2DATA` 模块，核心任务是让用户通过自然语言直接从数据源中获取所需数据，并为后续“输出为接口”或“继续做数据研究”提供中间工作台。

### Brand Personality
整体气质应当是：专业、冷静、可信。

界面不追求营销感，而是强调工作台属性，让用户在复杂配置和多步骤调试中依然能快速理解当前状态、下一步动作和关键结果。

### Aesthetic Direction
默认采用浅色主题，延续当前仓库已经建立的 Studio 风格 UI 语言：

- 简洁
- 模块清晰
- 面向持续扩展

视觉上保持轻量、克制、有秩序的分栏工作台结构，优先保证信息架构和功能边界清楚，避免花哨的装饰、过强的动效和高干扰配色。

### Design Principles
1. 工作流优先：页面首先服务于“选择资源 -> 发起意图 -> 查看 SQL -> 校验数据”这一主链路。
2. 模块边界清晰：不同能力区块必须能被快速识别，避免把对话、SQL、数据结果混成单一面板。
3. 保持可扩展：布局和组件设计要为后续增加辅助能力、更多数据源类型和输出能力预留空间。
4. 结果可验证：所有 AI 产物都要落到可见、可检查、可复用的 SQL 与结果面板中。
5. 克制而稳定：沿用现有轻量 Studio 风格，保持浅色主题、清晰层级和低干扰反馈。
